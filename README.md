Тестовое задание без использования Python  

Необходимо написать комплект powershell-скриптов, последовательно выполняющие следующие операции:  

1. Определять системный раздел и переразмечать его так, чтобы в его конце оказался валидный загрузочный EFI-раздел с размером, который можно задать параметром при запуске агента и стандартным размером в 10GB.  

2. Сжать системный раздел на максимально возможное пространство.  

3. Проверить результирующее НЕ аллоцированное место на накопителе данных с системным разделом для проверки результата сжатия.  Сжатого места должно быть не менее другого задаваемого при запуске параметра , со стандартным размером в 20 GB. В случае если места меньше -- операции необходимо откатить.  

 В случае ошибки на любом этапе скрипты отменять изменения на диске. Логи ошибки нужно выводить в консоль, а также при указанной опции записывать в файл передаваемый в аргументы.  

Большим плюсом будет реализация в виде программы-агента на Python.  

Параметры для запуска:  
-EfiSizeGB - Размер Efi раздела  
-MinUnallocatedGB - Минимальное освобожденное место после сжатия системного диска  
-EnableLog - Включение лога  
-LogFile - Путь до LOG файла  

.\TestDiskResize.ps1 -EfiSizeGB 10 -MinUnallocatedGB 20 -EnableLog -LogFile "C:\Temp\DiskResize.log"
